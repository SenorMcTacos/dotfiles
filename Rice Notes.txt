Hyprland

UWSM (Universal Wayland Session Manager)
Provides a cleaner exit out of the environment, by wrapping everything together
Requries the user to start processes with uwsm app --, which places apps in "slices"
These slices are then closed and managed by uwsm
3 slices, background, app, Session
background are low priority, session are high priority bg interactables, and app is main applications (firefox, games)
uwsm app -s (a|b|s) -- (app) # this is how you start stuff
systemctl --user enable (app) is used over uwsm 
ALSO NEED TO MOVE environment variable definitions in hyprland -> /uwsm/env
ALL EXPORTS ARE IN THE ENV FILE,  ~/.config/uwsm/env-hyprland and ~/.config/uwsm/env

ALSO TO DISABLE POWER SAVING FOR WIFI CARD
Recommended Method (NetworkManager)
If you are using NetworkManager (common in Ubuntu, Mint, Fedora, etc.), the most reliable method is to create or modify a configuration file: 

    Open or create the file /etc/NetworkManager/conf.d/default-wifi-powersave-on.conf using a text editor with root privileges:
    bash

sudo nano /etc/NetworkManager/conf.d/default-wifi-powersave-on.conf

Add or modify the following lines:
ini

[connection]
wifi.powersave = 2

(Value 2 means disable, 3 means enable).
Save the file and exit the editor.
Restart NetworkManager for the changes to take effect:
bash

sudo systemctl restart NetworkManager
